# 新增模块学习计划（未学过内容）

## 适用范围

以下模块你当前属于“未系统学习”，因此归为学习计划而不是复习计划：

- [[测试体系与质量保障]]
- [[应用安全与合规]]
- [[工程化交付与发布治理]]
- [[API设计与契约治理]]
- [[AI工程化与Java落地]]

## 学习原则

- 先理解概念，再做最小实践，再形成面试输出。
- 每个模块至少产出 1 份“可复述文档”和 1 个“可演示样例”。
- 学习完成后，再合并进 `[[复习分层导航]]` 的 `P1/P0`。

## 4周学习节奏（建议）

### 第1周：测试体系与API治理
- Day 1：测试分层与测试金字塔（单测/集成/端到端）。
- Day 2：JUnit5 + Mockito 常用套路。
- Day 3：Testcontainers + WireMock 实战。
- Day 4：契约测试（Pact）与接口兼容策略。
- Day 5：API 设计规范（错误码、幂等、版本化）。
- Day 6：OpenAPI 契约治理与变更评审流程。
- Day 7：周复盘（输出：测试策略文档 + API规范清单）。

### 第2周：安全与合规
- Day 8：Spring Security 认证授权主线。
- Day 9：OAuth2 Resource Server + JWT。
- Day 10：OWASP API Top 10 风险场景。
- Day 11：敏感数据分级、脱敏、审计。
- Day 12：依赖漏洞治理与 SBOM（CycloneDX）。
- Day 13：安全上线检查清单模板。
- Day 14：周复盘（输出：安全基线与应急流程）。

### 第3周：工程化交付与发布治理
- Day 15：CI/CD 流水线分层与门禁。
- Day 16：发布策略（滚动/蓝绿/灰度）对比。
- Day 17：Feature Flag 与回滚机制。
- Day 18：Kubernetes 探针/HPA 基础参数。
- Day 19：Helm 参数化配置与环境隔离。
- Day 20：DORA 指标与交付效能诊断。
- Day 21：周复盘（输出：发布SOP + 回滚预案）。

### 第4周：AI工程化与Java落地
- Day 22：LLM 基础参数与 Prompt 基础模式。
- Day 23：Tool Calling 与结构化输出。
- Day 24：Java 接入（Spring AI/LangChain4j）与治理。
- Day 25：RAG 链路（分块/检索/重排/更新）。
- Day 26：AI 可观测（延迟/成本/质量）与评估。
- Day 27：AI 安全（注入、越权、脱敏）与审计。
- Day 28：周复盘（输出：AI 最小可用 Demo 方案）。

## 验收标准

- 输出 5 份专题总结（每个模块 1 份）。
- 输出 3 个最小可用样例（测试、发布、AI 各 1 个）。
- 能完成 1 次“新增模块专题面试”模拟问答（30-45 分钟）。
