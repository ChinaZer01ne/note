# API设计与契约治理

## 定位

资深开发需要保证接口可演进、可兼容、可治理，避免跨团队协作成本失控。

## P0 必掌握

### 1. API 设计原则
- 资源建模、语义化命名、统一分页与排序。
- 错误码体系与错误信息规范。
- 幂等接口设计（重试安全、去重机制）。

### 2. 版本与兼容策略
- 向后兼容优先（新增字段优先、谨慎删除/重命名）。
- 版本策略（URI/Header/媒体类型）。
- 废弃流程：公告、过渡期、监控、下线。

### 3. 契约优先
- OpenAPI 文档即契约。
- 自动校验请求/响应契约一致性。
- 变更评审机制（Breaking Change 必须审批）。

## P1 进阶

### 1. 跨协议治理
- REST 与 gRPC 的选型边界。
- 网关统一治理（鉴权、限流、审计、路由）。
- 对外 API 与内部 API 的边界隔离。

### 2. 可观测与容量
- 接口级指标：QPS、P95/P99、错误率、超时率。
- 热点接口治理与配额管理。

## 面试输出模板

- 如何设计一个“可长期演进”的对外 API？
- 如何防止接口变更引发大面积调用方故障？
- REST 与 gRPC 在你项目中的分工策略是什么？
