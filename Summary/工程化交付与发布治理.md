# 工程化交付与发布治理

## 定位

资深开发不仅负责写代码，还需要保障“可预测交付”和“可控发布”。

## P0 必掌握

### 1. CI/CD 流水线设计
- 提交即验证：编译、单测、静态扫描、制品构建。
- 分支策略与发布策略（主干开发/发布分支）。
- 失败快速反馈与回滚机制。

### 2. 发布策略
- 滚动发布、蓝绿发布、金丝雀发布的适用场景。
- Feature Flag（特性开关）与灰度范围控制。
- 发布前检查清单：容量、依赖、回滚预案、监控看板。

### 3. 云原生部署要点
- Docker 镜像分层与最小化。
- Kubernetes 探针、资源配额、HPA 基础配置。
- Helm Chart 参数化与多环境配置管理。

## P1 进阶

### 1. 交付效能指标
- DORA 指标：部署频率、变更前置时间、变更失败率、恢复时间。
- 以指标驱动瓶颈治理，而不是只做统计报表。

### 2. 成本与容量协同
- 基于流量峰谷做弹性策略。
- 资源利用率与性能目标平衡（成本可观测）。
- FinOps 基础：为业务目标做成本优化。

## 面试输出模板

- 你如何设计一条“可回滚、可观测、可审计”的发布流程？
- 如何避免灰度发布引入双写/兼容性问题？
- 线上故障时发布系统如何支持快速止损？
