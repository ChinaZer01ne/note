# 应用安全与合规

## 定位

资深 Java 开发需要具备“默认安全”的设计能力，覆盖鉴权、数据保护、依赖安全与审计。

## P0 必掌握

### 1. 认证与授权
- Spring Security 基础模型（认证、授权、过滤器链）。
- OAuth2 Resource Server + JWT 校验流程。
- RBAC/ABAC 权限模型与最小权限原则。

### 2. API 安全基线
- OWASP API Top 10 常见风险与治理手段。
- 鉴权绕过、越权访问、重放攻击、接口刷量防护。
- 防重放与幂等设计（nonce、timestamp、幂等键）。

### 3. 数据安全
- 敏感数据分级与脱敏展示。
- 传输加密（TLS）与存储加密（字段级/磁盘级）。
- 密钥与凭证管理（避免明文配置）。

## P1 进阶

### 1. 供应链安全
- 依赖漏洞治理（SCA、升级策略、风险评估）。
- SBOM（CycloneDX）生成与归档。
- 基础镜像与运行时漏洞扫描。

### 2. 安全运营
- 安全审计日志设计（谁在什么时候做了什么）。
- 异常行为检测与告警闭环。
- 安全事件应急流程与复盘机制。

## 面试输出模板

- 如何在微服务体系落地统一认证与细粒度授权？
- 如何保证“上线速度”和“安全门禁”不冲突？
- 你们如何做依赖漏洞治理与紧急修复？
