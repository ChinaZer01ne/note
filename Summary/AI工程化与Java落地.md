# AI工程化与Java落地（新时代工程师必修）

## 学习目标

不追求“训练大模型”，而是掌握**把AI能力稳定接入业务系统**的工程能力。

## 必须掌握（P0）

### 1. LLM应用基础
- Token、上下文窗口、温度、TopP 等核心参数含义与取舍。
- Prompt 基础模式：指令、角色、约束、示例（few-shot）。
- 结构化输出（JSON Schema）与失败重试策略。
- Function Calling / Tool Calling 的工作流。

### 2. Java集成落地
- Spring AI 或 LangChain4j 的基础接入方式。
- 超时、重试、限流、熔断、降级（不可把AI调用当普通RPC处理）。
- 流式输出、异步调用、请求幂等与审计日志。

### 3. RAG（检索增强生成）
- 分块策略（chunk size/overlap）与召回率影响。
- 向量检索 + 重排（rerank）基本链路。
- 知识库更新策略（增量更新、失效与版本管理）。
- 幻觉治理：拒答策略、引用来源、置信度阈值。

### 4. 评估与可观测
- 线上指标：延迟、成功率、Token消耗、单位请求成本。
- 质量评估：正确性、相关性、一致性、可解释性。
- A/B 实验与回归评测（版本迭代必须可对比）。

### 5. 安全与合规
- Prompt Injection、越权调用、数据泄露风险。
- 输入/输出过滤与敏感信息脱敏。
- 模型调用审计、数据最小化与权限边界。

## 建议掌握（P1）

### 1. 多模型路由与成本优化
- 按任务复杂度做模型分级（小模型优先、复杂请求升级）。
- 结果缓存策略（语义缓存与失效策略）。
- 预算控制与降级开关。

### 2. Agent基础实践
- 多工具编排、任务分解、状态管理。
- 人工确认节点（Human-in-the-loop）与风险兜底。

## 面试输出模板

- 你如何把“AI问答”从Demo做成生产可用服务？
- 如何平衡“回答质量、响应延迟、调用成本”？
- 如何防止模型生成错误信息并影响核心业务？
