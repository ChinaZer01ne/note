# 测试体系与质量保障

## 定位

面向资深 Java 开发，目标是建立“可持续交付”的测试能力，而不是只会写单测。

## P0 必掌握

### 1. 测试分层与测试金字塔
- 单元测试：聚焦业务规则，快、稳定、可重复。
- 集成测试：验证模块协作（DB/MQ/Redis）。
- 接口/端到端测试：验证关键业务链路。
- 识别反模式：测试过慢、过度依赖外部环境、断言过弱。

### 2. Java 主流测试栈
- JUnit 5：生命周期、参数化测试、分组执行。
- Mockito：Mock/Spy、行为校验、避免过度 Mock。
- AssertJ/Hamcrest：可读断言。

### 3. 依赖隔离与可重复环境
- Testcontainers：MySQL/Redis/Kafka 集成测试环境一致化。
- WireMock：外部 HTTP 依赖模拟。
- 数据准备与回滚策略（测试数据污染治理）。

### 4. 契约测试与兼容性
- Pact 思路：消费者驱动契约。
- API 向后兼容策略：字段新增、字段废弃、错误码兼容。

## P1 进阶

### 1. 质量门禁
- 单测覆盖率阈值（关注核心路径而非盲目追高）。
- 静态扫描 + 代码规范 + 架构规则检查。
- PR 必过门禁策略与失败快速反馈。

### 2. 非功能测试
- 性能测试：延迟、吞吐、资源消耗。
- 稳定性测试：故障注入、限流降级验证。
- 混沌测试：网络抖动、节点异常、依赖超时场景。

## 面试输出模板

- 你如何设计一个“下单-支付-发货”链路的测试策略？
- 如何平衡“测试覆盖率”与“研发效率”？
- 如何通过契约测试避免微服务升级导致联调事故？
