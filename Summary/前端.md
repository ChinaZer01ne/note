### 样式层叠

样式冲突的权重优先级从**高 → 低**：
1. **!important** （无敌）
2. **行内样式 style=""** （1000）
3. **ID 选择器 #header** （100）
4. **类选择器 .box/ 属性选择器 [type] / 伪类 :hover** （10）
5. **元素选择器 div /li/p** （1）
6. **通配符 * / 继承样式** （0）
#### 计算规则

**权重相加，大的赢；相等，后面覆盖前面。不冲突样式 ,全部合并生效**

例子：

```css
div.box        /* 1 + 10 = 11 */
#header .item  /* 100 + 10 = 110 */
ul > li.active /* 1 + 1 + 10 = 12 */
```


### 继承

* 子元素自动拿到父元素的文字样式
* 文字属性可继承，布局属性不继承
* 继承权重 = 0，最最弱，随便都能覆盖

### CSS属性值的计算过程

* 收集样式
	* 浏览器默认样式
	- 你写的 CSS 样式
	- 行内样式
* 层叠（权重 + 顺序）
	* 比权重
	- 权重相同，后面覆盖前面
* 继承 / 默认值
	* 优先继承 → 继承不到就用默认值（initial）
* 转成绝对数值（px）
	* 相对单位转成绝对像素

> a元素为什么不继承？因为对于a标签浏览器有默认的颜色，后面就不继承了。除非设置`color: inherit`强制继承。

### 盒子模型

#### 行盒和块盒

* 行盒（内联元素）：元素只占本身内容需要的大小，挨个往后排。
	* 高度和宽度对内联元素没有影响；**`margin`和`padding`只有水平方向的作用，不能设置 `width`、`height`**。
	* `a`、`strong`、`img`、`em`、`button`等等。
	* `display:inline`
* 块盒（块级元素）：每个元素占一行，垂直堆砌。
	* **可以设置 `margin`、`padding` 四个方向，可以设置 `width`、`height`**
	* `body`、`main`、`header`、`footer`、`section`、`nav`、`aside`、`div`、`h1`-`h6`、`p`、`ul`、`ol`、`li`等等。
	* `display:block`
* 行块盒（内联块框（`inline-block boxes`））
	* 素只占本身内容需要的大小，挨个往后排，能设置宽高和margin和pading四个方向属性。
	* `img`其实具备这种特性，它是可替换元素。

> 虽然`img`、`button`等是`display:inline`，但他们属于行内可替换元素，可以设置宽高。

#### 盒子的组成部分
* margin：盒子之间的距离，外边距
	* margin-left、margin-right、margin-top、padding-bottom
* padding：填充，盒子边框到内容的距离，内边距
	* padding-left、padding-right、padding-top、padding-bottom
* boder：边框
	* boder-witdh、boder-style、boder-color
* content：内容
	* width：设置内容的宽度
	* height：设置内容的高度

> * 内容盒（content-box）:content
> * 填充盒（padding-box）：填充+内容
> * 边框盒（boder-box）：边框+填充+内容

### 布局

#### 常规流布局

* 所有元素默认情况下都属于常规流布局。
* 规则：
	* 块盒独占一行，行盒水平依次排列。
	* 每个块盒的总宽度，必须刚好等于包含块的宽度。
		* 常规流中，块盒在其包含块中居中的方式，可以定宽，然后左右margin设置为auto。
	* 每个块盒垂直方向上的auto值。
		* `height:auto`：适应内容的高度
		* `margin:auto`：表示0
	* 百分比取值
		* padding、宽、margin可以取百分比
		* 以上所有百分比取值是相对于包含块的**宽度**
		* 高度的百分比是相对于父元素高度，父元素如果没有设置高度，则子元素的百分比高度设置无效。
	* 上下外边距合并（所谓塌陷）
		* 两个常规流块盒，上下外边距相邻，会进行合并，取最大值。



> 包含块（containing block）：每个盒子都有它的包含块，包含块决定了盒子的排列区域。绝大部分情况，**盒子的包含块，就是其父元素的内容盒**。

#### 浮动

##### 应用场景

* 文字环绕
* 横向排列

##### 浮动的基本特点

* `float:left`：左浮动，元素相对于父元素内容盒靠上靠左
* `float:right`：右浮动，元素相对于父元素内容盒靠上靠右
* 宽度为auto时，适应内容宽度
* 高度为auto时，适应内容宽度
* margin为auto时，表示0
* 边框、内边距、百分比设置与常规流一样
* 左浮动盒子靠上靠左依次排列
* 右浮动盒子靠上靠右依次排列
* 浮动盒子在包含块中排列时，会避开常规流块盒
* 常规流块盒在排列时，无视浮动盒子
* 行盒在排列的时候，会避开浮动盒子（文字环绕）
* 不会发生外边距合并

##### 高度坍塌

常规流盒子的自动高度，在计算时，不会考虑浮动盒子。也就是说浮动盒子并不能将常规流盒子撑开。

如何解决？

```css
{
	/*
	   默认值：none
	   left:清除左浮动，该元素必须出现在前面所有左浮动盒子的下方 
	   right:清除右浮动，该元素必须出现在前面所有右浮动盒子的下方
	   both:清除左右浮动，该元素必须出现在前面所有浮动盒子的下方
	*/
	clear: both
}
```



> 元素浮动后，display会变成块盒。

> 关于文字，浏览器会默认用行盒包裹。

#### 定位

##### position属性

* 默认值：static，静态定位（不定位）
* relative：相对定位
* absolute：绝对定位
* fixed：固定定位

> absolute和fixed定位会让元素脱离文档流
> 文档流中的元素摆放时，会忽略脱离了文档流的元素
> 文档流中元素计算自动高度时，会忽略脱离了文档流的元素

##### 相对定位

```css
position:relative
```

不会导致元素脱离文档流，只是让元素在原来位置上进行偏移。

可以通过一下四个CSS属性设置其位置：
* left：离原本位置向右偏移的像素
* right：离原本位置向左偏移的像素
* top：离原本位置向下偏移的像素
* bottom：离原本位置向上偏移的像素

盒子的偏移不会对其他盒子造成影响。

> 相对定位，一般用来给绝对定位提供包含块的。

##### 绝对定位

```css
position:absolute
```

* 宽高为auto，适应内容
* 包含块变化：找祖先元素中第一个定位元素，该元素的填充盒为其包含块，如果找不到包含块，那包含块是整个网页。通过给其父元素`position:relative`就可以设置其包含块。
* 绝对定位是相对于其包含块的位置，通过设置left、top，距离包含块左边和上边的距离，或者设置right、bottom，距离包含块右边和下边的距离，来定位其位置。

> 在绝对定位和固定定位体系下，先定宽定高，left、right、top、bottom设置为0，margin设置为auto时，会自动吸收剩余空间，达到居中的效果。
> 
> 绝对定位元素一定是块盒
> 绝对定位元素不是浮动
> 没有外边距合并
##### 固定定位

```css
position:fixed
```

* 除了包含块不一样，其他和绝对定位一样
* 固定定位的包含块：固定为视口（浏览器的可视窗口）

> 在绝对定位和固定定位体系下，先定宽定高，left、right、top、bottom设置为0，margin设置为auto时，会自动吸收剩余空间，达到居中的效果。
> 
> 固定定位元素一定是块盒
> 固定定位元素不是浮动
>  没有外边距合并

##### 多个定位元素重叠时

* 通过设置`z-index`属性，可以完成元素层级的设置，该值越大，越靠近用户（在上层）
* 只有定位元素设置`z-index`有效
* `z-index`可以是负数。常规流元素和浮动元素可以覆盖`z-index`为负数的元素


* 默认定位
	* normal flow
	* `position:relative`
* 绝对定位
	* 不影响其他元素
	* out of flow
	* 默认是相对于viewport的定位。通过设置其第一个父元素的定位为relative，可以设置该元素绝对定位的相对容器。
	* `position:absolute`


* `Float Layouts`
	* 虽然还在用，但是过时的
	* `float:left`、`float:right`
	* 浮动元素会out of flow，但是文本和内联元素会环绕在浮动元素周围（区别于绝对定位的不影响其他元素的特性）
	* 由于out of flow，容器不再调整元素的高度，高度会坍缩成0，通过清除浮动`clear:both`来解决这个问题。
* `Flex Box`
	* 适合一维布局
	* `display:flex`
* `CSS Grid`
	* 适合二维布局

#### Flex Box

* `Flex Container`
	* `gap:0 | <length>`
		* 不使用margin，在两个item之间创建空格
	* `justify-content:flex-start | flex-end | center | space-between | space-around | space-evenly`
		* 水平方向
	* `align-items:stretch | flex-start | flex-end | center | baseline`
		* 垂直方向
	* `flex-direction:row | row-reverse | column | column-reverse`
		* main axis
	* `flex-wrap:nowrap | wrap | wrap-reverse
		* wrap into new line`
	* `align-content:stretch | flex-start | flex-end | center | space-between | space-around`
		* `flex-wrap:wrap`
* `Flex Items`
	* `align-self:auto | stretch | flex-start | flex-end | center | baseline`
		* 覆盖`align-items`
	* `flex-grow:0 | <integer>`
	* `flex-shrink:1 | <integer>`
	* `flex-basis:auto | <length>`
	* `flex:0 1 auto | <int> <int> <length>`
	* `order:0 | <integer>`

#### Css Gird
* `Grid Container`
	* `grid-template-rows:<track size>*`
	* `grid-template-columns:<track size>*`
	* `gap:0 | <length>`
		* `row-gap:0 | <length>`
		* `colum-gap:0 | <length>`
	* `justify-items:stretch | start | end | center`
	* `align-items:stretch | start | end | center`
	* `justify-content:stretch |start | center | end`
	* `align-content:stretch |start | center | end`
* `Grid Items`
	* `grid-column:<start line> / <end line> | span <number>`
	* `grid-row:<start line> / <end line> | span <number>`
	* `just-self:stretch | start | end | center`
	* `align-self:stretch | start | end | center`
> 在CSS Grid布局中，使用`fr`而不是`px`可以让元素长度或宽度动态变化，比如`grid-template-columns: 2fr 1fr 1fr 1fr`；当指定值是`2fr`的时候，表示长度或宽度是`1fr`的两倍。`fr`更类似是百分比的概念。如果配置`grid-template-columns: 2fr 1fr 1fr auto`，那么第四列只会占用自己需要的空间。
> 要理解`grid-column`和`grid-row`需要知道，在Css Grid中是按照行和列进行划线，起始从1开始。`grid-column:1/2`表示元素的列在第一道线和第二道线之间。


#### less

LESS 是 CSS 预处理器，扩展了 CSS 的语法（支持变量、嵌套、混合、函数等），让样式编写更高效。
✅ 建议装：如果你的项目需要写复杂样式、追求样式代码的可维护性（比如统一主题色、复用样式片段）；
❌ 可以不装：如果只是写简单的原生 CSS，或用 CSS Modules、Tailwind CSS 替代。

>安装 less 后，Vite 会自动识别 `.less` 文件，无需额外配置（Vue 项目直接用）。

#### vue-router

Vue 官方的路由工具，实现单页应用（SPA）的页面跳转（比如从首页跳转到详情页、登录页），无需刷新浏览器。没有 vue-router 的话，Vue 项目只能是单个页面，无法实现多页面切换。
✅ 必须装：只要你的项目是「多页面应用」（比如有首页、列表页、详情页）；
❌ 可以不装：仅做单个组件 / 静态页面展示（比如纯展示的官网单页）。

#### element-plus

Vue3 官方推荐的桌面端 UI 组件库，提供现成的按钮、表单、表格、弹窗、导航等组件，无需自己从零开发。
✅ 建议装：开发后台管理系统、企业官网等项目，快速搭建界面，节省大量样式 / 交互开发时间；
❌ 可以不装：如果需要高度定制化的 UI（比如设计要求独特的移动端页面），或用其他组件库（如 Naive UI、Ant Design Vue）。

#### element-plus/icon-vue

Element Plus 的配套图标库，提供几百个常用图标（如搜索、删除、返回、加载），需配合 Element Plus 使用。
✅ 必须装：如果装了 Element Plus，且需要用它的内置图标；
❌ 无需装：不用 Element Plus，或用其他图标库（如 IconPark、Font Awesome）。

#### vue-router路由配置

配置组件：
```javascript
import { createRouter, createWebHashHistory } from "vue-router";

const routes = [
	// 根路径映射到某一个组件，Main.vue是你自己定义的组件
    {
        path: '/',    // 访问路径（根路径，即网站首页）
        name: 'main',    // 路由的唯一名称（可选，但建议加，方便后续跳转）
        component:()=>import("@/views/Main.vue"),    // 路径对应的页面组件
        children: [ // 子路由
            {
                path: '/home',
                name: 'home',
                component:()=>import("@/views/Home.vue")
            }
        ]
    },
    // 新增其他页面路由 { path: '/about', name: 'about', component:()=>import("@/views/About.vue") },
];

const router = createRouter({
    history:createWebHashHistory(),// 启用 Hash 历史模式
    routes,// 传入上面定义的路由规则
});

export default router;//导出路由实例
```

main.js挂在路由组件：
```javascript
const app = createApp(App) // 把路由实例挂载到 Vue 应用上 
app.use(router) 
app.mount('#app')
```

App.vue 中使用组件：
```javascript

</script>

<template>
	<router-view></router-view>
</template>

<style scoped>
</style>
```

官网：https://pinia.vuejs.org/zh/introduction.html


#### 引入pinia

```javascript

```

#### 使用pinia

创建`stores/index.js`如下

```javascript
import { defineStore } from 'pinia'
import { ref } from 'vue'
// isCollapse是组件之间的共享变量
function initState() {
    return {
        isCollapse: true
    }
}

export const useAllDataStore = defineStore('allData', () => {
  const state = ref(initState())
  return { state, }
})

```

读共享变量（某个组件）

```html
<el-aside :width="width">
	<!-- isCollapse控制折叠效果 -->
	<el-menu :collapse="isCollapse"></el-menu>
</el-aside>
```
```javascript
import { useAllDataStore } from "@/stores";
const store = useAllDataStore()
const isCollapse = computed(()=>store.state.isCollapse)
const width = computed(()=>store.state.isCollapse ? '64px' : '180px')
```

写共享变量（另一个个组件）

```html
<el-button size="small" @click="handleCollapse"></el-button>
```
```javascript
import { useAllDataStore } from "@/stores";
const store = useAllDataStore()
function handleCollapse() {
	store.state.isCollapse = !store.state.isCollapse
}
```

#### 对axios进行二次封装

##### 学会环境隔离配置

创建`config/index.js`

```javascript
const env = import.meta.env.MODE || 'prod'

const EnvConfig = {
    development:{
        baseApi:"dev/api",
        mockApi:"http://dev-mock/api"
    },
    test:{
        baseApi:"/test/api",
        mockApi:"http://test-mock/api"
    },
    prod:{
        baseApi:"/pro/api",
        mockApi:"http://pro-mock/api"
    }
}

export default {
    env,
    ...EnvConfig[env],
    mock:true
}
```

创建`request.js`

```javascript
import axios from "axios";
import { ElMessage } from "element-plus";
import config from "@/config";

const NETWORK_MSG = "网络错误..."
const service = axios.create({
    baseURL:config.baseApi
});

// 添加请求拦截器
service.interceptors.request.use(function (config) {
    // 在发送请求之前做些什么
    return config;
  }, function (error) {
    // 对请求错误做些什么
    return Promise.reject(error);
  });

// 添加响应拦截器
service.interceptors.response.use((res) => {
    const {code, data, msg} = res.data
    if (code == 200) {
        return data
    } else {
        ElMessage.error(msg || NETWORK_MSG);
        return Promise.reject(msg || NETWORK_MSG);
    }
});

  
function request(options) {
    options.method = options.method || 'get'
    if (options.method.toLowerCase() === 'get') {
        options.params = options.data;
    }
    let isMock = config.mock;
    if (typeof options.mock != "undefined") {
        isMock = options.mock
    }

    if (config.env === 'prod') {
        service.defaults.baseURL = config.baseApi
    } else {
        service.defaults.baseURL = isMock ? config.mockApi : config.baseApi
    }

    return service(options)
}

export default request;
```

创建`api.js`

```javascript
import request from "./request";

export default {
    getTableData() {
        return request({
            url:"/home/getTableData",
            method:"get",
            mock:false
        })
    }
};
```

`main.js`下挂载`api`调用

```javascript
import api from "@/api/api";
app.config.globalProperties.$api = api;
```

在组建中使用axios的封装

